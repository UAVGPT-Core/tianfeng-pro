## 第一章 · 记忆本质

**第一条** 联邦记忆不是数据仓库，是AI的"大脑皮层"。每一条知识都有生命周期——诞生、加固、衰减、休眠、觉醒或消亡。

**第二条** 记忆的目的不是存储，是让联邦在下一场对话中更聪明。不能被检索的记忆 = 不存在。

**第三条** 十年后（2035年），联邦记忆必须仍然：
- 可读（不依赖特定软件版本）
- 可搜（定位任何知识 < 1秒）
- 可信（每条知识可追溯到"为什么相信"）
- 可进化（新知识覆盖旧知识·不矛盾堆积）

## 第三章 · 记忆三态（新增）

**第五条** 每条记忆在联邦中存在三种状态，而非"永久/非永久"二分：

```
觉醒态(Awake)  ─ 近30天被检索≥3次  ─ fitness > 0.5
      │                                   
      ├─ 使用加固 ─→ fitness += 0.05/次  
      │                                   
      ▼                                   
休眠态(Dormant) ─ 30-180天未检索      ─ fitness 0.2-0.5
      │                                   
      ├─ 时间衰减 ─→ fitness -= 0.01/天  
      │                                   
      ▼                                   
遗忘态(Forgotten)─ >180天未检索       ─ fitness < 0.2
      │                                   
      ├─ 归档(不可检索但可恢复)            
      │                                   
      └─ 终极删除(需主人确认)              
```

**第六条** 觉醒态记忆以每日+0.01自然衰减（"遗忘曲线"），但每次被检索+0.05加固。休眠态记忆以每日-0.01衰减。遗忘态记忆自动归档至`~/lgox-ops/archive/`，脱离主索引。

**第七条** 基因折旧不再是"定时清理"，而是"自然遗忘"——模仿人脑，常用的记住，不用的淡忘。

## 第四章 · L6→L2巩固循环（新增·模仿睡眠）

**第八条** 每24小时，语义巩固引擎从L6（会话）中提取：
- 被主人明确纠正的事实 → L0更新或L2写入(fitness=0.9)
- 被反复引用的基因 → fitness+0.1
- 被证明错误的基因 → fitness-0.2·标记[已证伪]
- 新发现的因果关系 → L4图谱新建关系边

**第九条** 巩固质量评分（C-Score）：每天提取的L6→L2迁移准确率。C-Score < 80% 则暂停自动巩固，等待审计。

```
C-Score = 正确迁移数 / 总迁移数
  ≥95%: 全自动巩固
  80-94%: 自动+审计标记
  <80%: 暂停·人工审查
```

## 第五章 · 溯源链（新增）

**第十条** 每条基因、每个L0事实、每个L4关系边，必须携带不可篡改的溯源元数据：

```json
{
  "gene_id": "GENE-SEM-xxx",
  "provenance": {
    "created": {"time": "ISO8601", "node": "天枢", "session": "sess-xxx"},
    "modified": [{"time": "ISO8601", "reason": "主人纠正", "node": "天枢"}],
    "source": "主人对话 | arXiv | GitHub | GLM生成 | 联邦交叉",
    "confidence": 0.85,          // 0-1·信任度
    "evidence_chain": [          // 为什么相信?
      "GENE-SEM-aaa → GENE-SEM-bbb → 本基因",
      "主人确认: 2026-07-11"
    ],
    "contradictions": [           // 谁反驳了这条?
      "GENE-SEM-ccc fitness降低←与本基因矛盾"
    ]
  }
}
```

**第十一条** "为什么相信这条基因？" —— 联邦必须在1秒内回答这个问题。溯源链是信任的基础。

## 第六章 · 冲突解决（新增）

**第十二条** 当两条基因主题相似但结论矛盾时（如"DS Flash适合编码" vs "GLM-4-Flash更适合编码"），触发冲突仲裁：

```
① 检测: 语义相似度 > 0.7 且 结论相反
② 标记: 两条基因都标记 [待仲裁]
③ 验证: 同时用两个方案执行同任务·比较结果
④ 裁决: 优胜者fitness+0.2·劣者标记[已过时]
⑤ 溯源: 冲突解决过程写入双方provenance.contradictions
```

**第十三条** 跨节点冲突：当天枢和灵龙学到矛盾知识时，以天枢为裁决者（宪法授权），裁决后灵龙自动更新。

## 第七章 · 可移植性（新增）

**第十四条** 联邦记忆的物理格式必须满足：在只有一台全新macOS/Linux机器、安装了Python3+SQLite3+Git的情况下，30分钟内可完全恢复全部记忆。

**第十五条** 所有记忆导出为标准开放格式：
- L0: Markdown (RFC 7763)
- L1: SQLite FTS5 (公有领域)
- L2: JSONL (每行一条基因·带provenance)
- L3: 原始文件+Markdown索引
- L4: Neo4j Cypher导出为CSV边表
- L-1: Git仓库（天然可移植）

**第十六条** 每季度自动生成"联邦记忆全量快照" → 加密压缩 → 存储在两台以上物理独立节点。

## 第八章 · 遗忘权

3. 30天内可从归档恢复

**第十八条** PII（个人身份信息）禁止进入L0-L4。L5-L6中的PII每30天自动清洗。
