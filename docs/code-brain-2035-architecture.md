## 0. 核心洞察

到2035年，LLM模型已成为商品。决定代码生成质量的不再是"哪个模型更强"，而是：

> **基因记忆 × 多模型互审 × 自我对弈 = 代码能力指数增长**

## 2. 七层推理管线

```
L6 🧬 CRYSTALLIZE  固化层 — 每次代码交互→基因永久固化→全联邦同步
L5 🔄 SELF-PLAY    对弈层 — 自动出题→生成→审查→改进→纳基因 永动循环
L4 ✅ VERIFY       验证层 — 三模型交叉审查+编译+测试+安全+形式化
L3 🏗️ CODE-GEN     生成层 — 多模型并行生成→最优选择→基因上下文注入
L2 🧠 REASON       推理层 — 需求→规格→架构→设计模式→Trade-off分析
L1 📋 SPEC         规格层 — 模糊需求→结构化规格→基因检索→需求补全
L0 🔀 ROUTE        路由层 — 天工qwen2.5-coder/DS Pro/GLM/本地r1
```

### 各层详解

#### L0 路由层 ROUTE
- 任务分类: 简单代码(天工)→中等(DS Flash)→复杂(DS Pro)→架构(GLM)
- 成本感知: 优先免费本地模型，复杂才用云端Pro
- 并行分发: 关键任务可多模型并行

#### L1 规格层 SPEC
- 输入: 模糊需求("帮我写个爬虫")
- 输出: 结构化规格(功能·接口·约束·边界条件·测试用例)
- 基因注入: 从LGE检索类似需求的历史方案

#### L2 推理层 REASON
- 架构决策: 选择什么设计模式? 为什么?
- Trade-off分析: 性能vs可读性·复杂度vs可维护性
- 多模型思辨: 3模型各自提出方案→辩论→综合

#### L3 生成层 CODE-GEN
- 3模型并行生成(不同temperature)
- 最优选择: 编译通过率→测试覆盖率→复杂度→审查评分
- 基因模板注入: 已验证的代码模式直接复用

#### L4 验证层 VERIFY
- 编译验证: 代码必须能编译/运行
- 测试生成: 自动生成单元测试+边界测试
- 三模型交叉审查: 模型A生成→模型B审查→模型C终审
- 安全扫描: SQL注入·XSS·敏感信息泄露
- 复杂度检查: 大O·圈复杂度·依赖深度

#### L5 对弈层 SELF-PLAY
- 出题引擎: 从LGE基因库自动生成代码题目
- 解题→审查→改进→纳基因
- 难度自适应: 简单→中等→困难→极限
- 维度: 算法·数据结构·设计模式·系统设计·Bug修复

#### L6 固化层 CRYSTALLIZE
- 每个成功代码→semantic基因
- 每个Bug修复→procedural基因(含踩坑记录)
- 每个设计决策→episodic基因(含上下文)
- 全联邦同步: 灵龙学会→天枢→天工→地枢→全节点
